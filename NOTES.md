`AsyncNotifierProvider` is a better fit here because the state needs to change more than once: it can return cached data first, then later push a refreshed value without rebuilding the whole provider shape. A `FutureProvider` is great for one async computation, but it is less ergonomic when the feature also needs imperative methods like `refresh()` and background updates after the initial result. The cache-first strategy prevents the user from landing on an empty loading screen every time they reopen the app. It also reduces visible flicker and makes the experience feel resilient when the network is slow or temporarily unavailable.
